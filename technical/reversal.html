Reversal

Request that a previously initiated transaction is reversed. Commonly referred to as reversal or void.

A transaction is reversed via specifying the id of the transaction, which assumes you have access to the TransactionResponse for that transaction from the PaymentResponse. Even if the payment app malfunctioned and did not produce a response, FPS will have created a default declined response with the id that was received by the payment app.

TransactionResponse responseToReverse = paymentResponse.getTransactions().get(desiredTxnIndex).getPaymentAppResponse();
if (responseToReverse != null) {
    String reversalId = responseToReverse.getId();
}

Note the getPaymentAppResponse() call, which will always return a response from the payment application, if one was generated. A payment application is not necessarily called for a transaction. There may be responses that would have come from value added flow services paying off a portion (or all) of the amounts, such as loyalty points / rewards.

If you require additional data for your use case / scenario, please contact AEVI to discuss.

Create Request

Request reversalRequest = new Request("reversal");
reversalRequest.addAdditionalData("transactionId", reversalId);

Create Response

String transactionId = reversalRequest.getRequestData().getStringValue("transactionId");
...
Response reversalResponse = new Response(reversalRequest, <outcome>, "Outcome message");
// original transaction ID should be returned
reversalResponse.addAdditionalData("transactionId", transactionId);
// add other payment app specific transaction reference data  here

Parse Response

String reversedSuccessfully = reversalResponse.wasSuccessful();
